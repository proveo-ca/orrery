# Security Policy

`orrery` is a meta-repository: it holds Proveo's shared `_spec/` learning surface plus pointers
(Git submodules) to the application projects. The applications themselves live in their own
repositories — report issues specific to an app in that app's repo.

## Reporting a vulnerability

Please report security issues **privately** — do **not** open a public issue or pull request.

- **Preferred:** GitHub → the repository's **Security** tab → **Report a vulnerability**
  (this opens a private security advisory).
- We aim to acknowledge a report within a few business days and will coordinate a fix and a
  disclosure timeline with you.

When reporting, please include what you found, where (file / commit / submodule), how to
reproduce it, and the potential impact.

## Scope

- **In scope:** this repository — `_spec/` and its configuration.
- **Out of scope:** the individual project repositories (`aphelion`, `omnigent`, …); report
  vulnerabilities there.

Thank you for helping keep the project and its users safe.
